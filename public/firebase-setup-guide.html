Fix the Firebase Configuration Error

Current Issue

Error: Firebase: Error (auth/configuration-not-found)

This error occurs when Firebase Authentication is not properly configured in your Firebase project.

Quick Fix Steps

1

Go to Firebase Console

Visit your Firebase project:

Open Firebase Console ‚Üí
2

Enable Authentication

  • Click on Authentication in the left sidebar
  • Click Get started if you haven't already
  • Go to the Sign-in method tab
  • Enable Email/Password authentication
  • Make sure to enable both toggles (Email/Password and Email link)
3

Create Firestore Database

  • Click on Firestore Database in the left sidebar
  • Click Create database
  • Choose Start in test mode (for development)
  • Select your preferred location
  • Click Create
4

Configure Authorized Domains

  • In Authentication settings, go to Settings tab
  • Scroll to Authorized domains
  • Add these domains:
    localhost 127.0.0.1 [your-replit-url].repl.co (if using Replit)
5

Update Security Rules (Temporary for Testing)

In Firestore, go to Rules tab and update to:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if request.auth != null; } } }

Still Having Issues?

If you're still seeing errors:

  1. Check Browser Console: Look for specific error messages
  2. Verify API Keys: Ensure your .env file has the correct Firebase config
  3. Clear Browser Cache: Sometimes old configs get cached
  4. Check Network Tab: Look for 400/401 errors in browser DevTools

Common Error Messages:

  • auth/configuration-not-found - Authentication not enabled
  • auth/network-request-failed - Check internet connection
  • auth/invalid-api-key - Wrong API key in .env
  • auth/user-not-found - Need to create demo users first
